# Agent 时代个人生产力：从工具到协作

综合本库 4 篇资料，Agent 时代的个人生产力呈现三条交汇的主线：

## 1. 交互范式：从"操作软件"到"描述目标"
[[natural-language-task-driving|自然语言任务驱动]] 与 [[vibe-coding|Vibe Coding]] 是同一思想的两面——用户用语言描述目标，AI 直接执行或生成产物。这把"想法→结果"的路径压到一句话。

## 2. 能力组织：从"装软件"到"加载技能"
[[skills-system|Skills 技能系统]]（WorkBuddy）与扣子的行业技能包，都把可复用能力模块化、可发现、可组合，让单一工作台胜任多领域。

## 3. 协作形态：从"单人对工具"到"人与多 Agent 组队"
[[multi-agent-collaboration|多 Agent 协作]]（扣子）把任务拆给不同 Agent 分工，资产持续沉淀；[[ai-automation-pipeline|零代码自动化流水线]]（WorkBuddy）则把重复任务定时化。

## 4. 知识底座：让 AI 维护你的第二大脑
[[llm-wiki|LLM Wiki]] 方法指出：把资料交给常驻 LLM 消化成结构化 Wiki，实现 [[knowledge-compounding|知识复利]]，而非像传统 [[rag|RAG]] 那样"问完即忘"。这恰与上述生产力工具互补——工具负责"做"，Wiki 负责"记与连"。

## 实践建议
- 个人办公/文件自动化：优先 [[workbuddy|WorkBuddy]] + 自动化流水线 + 积分激励。
- 团队协作/AI 编程/视频：优先 [[coze|扣子]]。
- 长期学习沉淀：用 [[obsidian|Obsidian]] + [[agents-md|CLAUDE.md]] 跑一套 LLM Wiki。
