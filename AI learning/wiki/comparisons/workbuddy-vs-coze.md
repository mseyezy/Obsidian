# WorkBuddy vs 扣子 (Coze)

| 维度 | [[WorkBuddy]]（腾讯） | [[Coze|扣子]]（字节） |
|------|------|------|
| 核心定位 | 个人办公提效 + 本地文件操作 | 多 Agent 团队协作 + 创作 |
| 交互范式 | [[natural-language-task-driving|自然语言任务驱动]] | 多 Agent @协作 |
| 自动化 | [[ai-automation-pipeline|零代码流水线]]（签到/整理/周报） | 项目级多会话并行 |
| 编程 | 偏办公脚本/文件处理 | [[vibe-coding|Vibe Coding]]（网页/App/小程序） |
| 创作 | 较弱 | 强（长文/PPT/视频全流程） |
| 激励 | [[credits-points|积分 / 积分自由]] | 团队资产沉淀 |
| 本地能力 | 强（直接读写本机文件） | 桌面端可远程操作本地文件 |

## 结论
个人办公、文件处理、自动化签到/整理选 WorkBuddy；团队协作、AI 编程、视频创作选扣子。两者都体现了"Agent 时代工作台"的趋势。详见 [[ai-office-tools-overview|AI 办公工具总览]]。
